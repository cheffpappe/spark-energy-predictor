Precompute predictions to JSON for free-tier deploy

PySpark's JVM cannot fit in Render's 512 MB free tier, so materialize
every possible (hour, day, month, region) prediction once and serve
them from a lookup table. The runtime API is now pure Flask and runs
on the free plan.

- api/app.py: rewritten as lookup-backed Flask API (no PySpark)
- api/app_spark_legacy.py: original Spark-backed app, kept for precompute
- scripts/precompute.py: regenerates models/predictions.json via Spark
- models/predictions.json: 22,176 precomputed predictions (~514 KB)
- requirements.txt: slimmed to flask + gunicorn
- requirements-precompute.txt: Spark deps, only for regenerating the JSON
- render.yaml: Python native runtime on the free plan (no Docker)
- Dockerfile + .dockerignore: removed (no longer needed)
- README: updated with the new architecture and deploy flow
