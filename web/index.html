How it works

From raw grid data to a real-time MW forecast

An end-to-end Big Data pipeline: ingest PJM hourly loads, engineer temporal and regional features on a Spark cluster, train a Gradient Boosted Trees regressor with Spark ML, and serve low-latency predictions through a Flask REST API.

Data → Spark Processing → Feature Engineering → ML Model → Flask API → Web UI
01

Data Source

PJM Interconnection hourly load data across 11 regional grids (PJME, PJMW, AEP, DOM, COMED…).

02

Features

Hour (0–23), day-of-week (1–7, Monday = 1), month (1–12), and region index — vectorised with Spark ML.

03

Model

Gradient Boosted Trees regressor trained on Spark ML — captures seasonal + diurnal demand patterns.

04

Output

Predicted regional electricity demand (MW) generated in real time via a Flask API, powered by a Spark-trained machine learning model.

05

Why Apache Spark?

Energy datasets span billions of hourly records across years and regions. Spark parallelises ingestion, feature engineering and GBT training on distributed RDD/DataFrame workloads — delivering horizontal scalability, in-memory compute, fault tolerance and production-grade throughput that single-node Python cannot match.

Why this matters

From a grid forecast to real-world impact

Accurate short-horizon load prediction underpins nearly every lever utilities have for reliability, cost and decarbonisation. A Big Data pipeline like this one turns petabytes of raw telemetry into operational decisions.

01

Energy Demand Forecasting

Gives utilities hour-ahead visibility into regional load so generators can be dispatched, ramp rates optimised and reserve margins sized to actual demand — not worst-case estimates.

02

Grid Stability & Load Balancing

Reliable MW forecasts let system operators pre-position spinning reserves, balance inter-zonal transfers and prevent frequency excursions during peak events or renewable intermittency.

03

Cost Optimization

Reducing over-generation and expensive peaking-plant activations translates directly into lower wholesale prices, fewer emergency procurement contracts and smaller ancillary-service bills.

04

Smart Energy Management

Feeds demand-response platforms, battery-storage dispatch and EV-charging schedulers — enabling data-driven decarbonisation and truly intelligent grid management.

Model details

Under the hood

A concise technical summary of the algorithm, feature set and training data that produce every prediction above.

M

Model

Gradient Boosted Trees regressor (GBTRegressor) trained with Apache Spark ML — an ensemble of shallow decision trees that captures non-linear seasonal and diurnal demand patterns.

F

Features

Four engineered inputs — Hour (0–23), Day of week (1–7, Monday = 1), Month (1–12) and Region index — assembled into a feature vector via Spark ML's VectorAssembler.

D

Training data

Trained on the PJM Interconnection hourly energy dataset spanning 11 regional grids (PJME, PJMW, AEP, COMED, DOM, …). Millions of hourly load observations covering multi-year seasonality.
